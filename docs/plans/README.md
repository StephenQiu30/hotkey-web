# Plan 文档

本目录存放执行计划类文档。

## 适合放入

1. 阶段计划与任务拆解。
2. 任务编排和归档路线。
3. 里程碑、排期和交付顺序。
4. 风险、依赖和后续事项。

## 不适合放入

1. 产品需求原文。
2. 详细技术设计。
3. 验收报告或测试记录。
4. 仅服务当前执行的 todo 列表、临时进展记录或过程流水账。

## 命名建议

使用 `序号-主题-计划.md`，例如 `001-docs目录治理计划.md`。

## 状态门禁

只有 `status: approved`、`review_status: approved`、`execution_status: ready` 且前置依赖完成的 Plan 才能开工。文件、步骤、契约或验收变化后必须重新审核。
