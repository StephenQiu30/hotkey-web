# 内容归档阅读与工作台可视化设计

## 1. 设计目标

延续现有黑色 Vercel/Geist 风格、`.app-page`、`.panel`、`Button`、`Badge` 和 lucide 图标，不引入第二套设计系统。工作台要先解释“当前事件”，再展示证据和比较；内容详情要明确区分“来源已归档内容”与“访问原站”。

## 2. 工作台布局

页面居中且最大宽度与欢迎页/其他 dashboard 页一致。`xl` 及以上使用 `minmax(0,1fr) 360px`，以下自动单列；情报区在桌面端 sticky，不与主区内容再竞争第三列。

“证据” Tab 只显示全宽时间线，标题进入系统内 `/dashboard/contents/{id}`，“访问原站”单独打开 canonical URL；显示“当前展示 n / 共 m 条”，不隐藏只取前 8 条的事实。“事件热度对比”移入全宽“信号” Tab；全部为 0/缺失时显示“热度尚未计算”，不渲染残缺坐标标签。在没有事件-报告关联契约前，侧栏文案使用“最近报告”。

## 3. 内容详情和 Markdown

详情页通过生成客户端读取 `GET /api/v1/contents/{id}/document`。`ready` 时展示来源、时间、语言、canonical URL、内容类型提示和 Markdown；`not_captured` 时展示真实空态、原因和原站入口。页面不将 Feed 摘要称为完整原文。

复用 `react-markdown` + `remark-gfm`，不启用 `rehype-raw`。链接统一 `target=_blank`/`rel=noreferrer`，原始 HTML 不执行。阅读区限定约 72ch，表格可水平滚动，代码块换行与滚动二选一保证不撑破屏幕。

## 4. 打印 / 保存 PDF

`ready` 时按钮调用 `window.print()`，文案明确为“打印 / 保存 PDF”，不宣称服务端下载。print media 隐藏导航、toast、操作栏和空间装饰，保留标题、出处、canonical URL、时间与正文；`not_captured` 禁用该按钮。

## 5. 来源开关

只有 admin 的“新增来源”表单显示“保存 Feed 正文/摘要用于归档预览”开关，默认关闭。说明文案必须指出：只保存 Feed 实际提供的内容，不抓取原网页；启用前应确认来源条款。只有用户显式勾选时才提交 `config.allow_body_storage=true`。
