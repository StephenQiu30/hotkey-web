# 001 Web 工作台完整配置与查看

## 功能目标

Web 工作台是 V1 的完整配置和查看端。用户必须能在 Web 中完成账号登录、平台授权、主题配置、Git Vault 设置、邮件订阅、热点查看、事件详情、同步状态、失败处理、撤权和删除。

## 用户故事

1. 作为用户，我可以完成首次登录和产品配置。
2. 作为用户，我可以连接国内外平台账号并查看连接健康状态。
3. 作为用户，我可以创建主题、选择平台和设置关键词。
4. 作为用户，我可以查看热点事件、摘要、来源、评分和 Obsidian commit 链接。
5. 作为用户，我可以撤权、删除主题或删除账户。

## 范围

1. 登录和当前用户。
2. 平台授权连接列表、状态和操作。
3. 主题管理。
4. 事件列表和详情。
5. Obsidian Git 设置和同步状态。
6. 邮件日报/周报设置和发送历史。
7. 删除、撤权和失败状态展示。

## 非范围

1. 小程序配置端。
2. 团队协作。
3. 可编辑邮件模板。
4. 创作稿件生成。

## 状态模型

页面必须展示 server 返回的真实状态，不在前端臆造成功状态。关键状态包括：授权状态、主题状态、采集状态、事件摘要状态、Git 同步状态、邮件发送状态和删除清理状态。

## 契约影响

Web 只能通过 server OpenAPI 生成客户端类型。任何接口字段缺失必须先回到 server 修改 OpenAPI，不允许在 Web 手写临时类型绕过。

## SDD 要求

1. 先确认每个页面依赖的 OpenAPI。
2. 先定义关键用户路径和空态、加载态、错误态。
3. 先定义失败状态文案和可执行操作。
4. 先定义 E2E 选择器和测试数据准备方式。

## TDD 要求

1. 先写页面组件或路由级测试。
2. 先写 OpenAPI mock 返回各类状态的测试。
3. 先写权限失效和操作失败测试。
4. OpenAPI 变更后先生成客户端再实现 UI。

## E2E 验收门禁

1. Playwright 完成首次配置路径：登录、连接测试平台、创建主题、连接 Git、设置日报。
2. Playwright 查看事件详情：摘要、评分、来源和 commit 链接可见。
3. Playwright 覆盖平台失效、Git 同步失败和邮件失败状态。
4. Playwright 覆盖撤权和删除主题路径。
5. server E2E 数据必须驱动 Web E2E，不只 mock 前端页面。

## 数据与权限

1. Web 不存储平台 token 明文。
2. 敏感操作需要确认。
3. 删除和撤权操作必须展示进行中和最终状态。
