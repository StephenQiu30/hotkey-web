# 内容归档阅读与工作台可视化

## 背景

当前工作台在主区、460px 侧栏和 240px 图表之间形成三列竞争，零热度图只剩残缺标签。当前内容列表只能跳出原站，无法查看服务端已授权保存的内容。

## 用户故事

1. 作为分析者，我可以从证据时间线或采集内容列表进入系统内详情，阅读可用 Markdown，并保留原站入口。
2. 作为分析者，我可在浏览器打印或保存 PDF，且 PDF 中只包含文档本身。
3. 作为管理员，我可在新建来源时明确决定是否保存 Feed body，默认不保存。
4. 作为工作台用户，我在桌面、平板和手机上都能清晰理解事件、证据和热度，不被无意义图表干扰。

## 范围与非目标

范围为内容动态路由、GFM 预览、ready/not_captured/错误态、print media、来源开关、工作台双列/单列响应式、热度空态和证据计数。非目标为 canonical 抓取、服务端 PDF 下载、历史正文伪造或修改已发布来源。

## 验收门禁

- 应用只使用 OpenAPI 生成的 document 函数和类型。
- ready 渲染 GFM 且可打印；not_captured 不伪造 Markdown 且禁用 PDF；404/403/503 有可重试或返回的明确错误态。
- 原始 HTML 不执行，外链安全，长文/表格/代码不造成水平溢出。
- 开关只对 admin 显示、默认 false，未显式确认不得提交 true。
- 1440×900、1024×768、390×844 下布局正确；零热度不渲染空图；证据显示 n/m 和内部/原站双入口。
- `npm run test:unit && npm run typecheck && npm run build` 通过。
